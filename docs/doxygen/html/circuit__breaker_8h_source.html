15// ---------------------------------------------------------------------------
16// CircuitBreaker: per-replica health guard using the standard three-state
17// circuit breaker pattern (Nygard, "Release It!").
18//
19// State machine:
20//
21// CLOSED --[error_rate > threshold]--> OPEN
22// OPEN --[cooldown elapsed]--------> HALF_OPEN
23// HALF_OPEN --[probe succeeds]--------> CLOSED
24// HALF_OPEN --[probe fails]-----------> OPEN
25//
26// States explained:
27// CLOSED: Normal operation. All requests are allowed through.
28// Outcomes are recorded in a sliding window. If the failure
29// rate within the window exceeds error_threshold, the circuit
30// transitions to OPEN.
31//
32// OPEN: The replica is considered degraded. All requests are blocked
33// (AllowRequest returns false) for cooldown_ms milliseconds.
34// After the cooldown, the next AllowRequest call transitions
35// to HALF_OPEN and lets one probe through.
36//
37// HALF_OPEN: Exactly one "probe" request is in flight. If it succeeds,
38// the circuit returns to CLOSED (the replica has recovered).
39// If it fails, the circuit reopens (back to OPEN) with a
40// fresh cooldown timer.
41//
42// Why this complements gossip (SWIM):
43// Gossip detects binary failure (crash, network partition) via ping/ack
44// timeouts. But a replica can be alive at the gossip level while still
45// failing at the application level (e.g., OOM on large prompts, GPU
46// errors, model loading issues). The circuit breaker detects this
47// "gray failure" by tracking actual request outcomes, and temporarily
48// removes the replica from the routing pool without waiting for gossip
49// to declare it DEAD.
50//
51// Sliding window:
52// outcomes_ is a bounded deque of the last window_size request results
53// (true = success, false = failure). The window provides recency: old
54// failures are forgotten as new outcomes push them out, so a replica
55// that recovers will naturally close its circuit.
56// ---------------------------------------------------------------------------
