21// ---------------------------------------------------------------------------
22// GatewayServer: the client-facing gRPC endpoint for inference requests.
23//
24// Role -- streaming reverse proxy:
25// The gateway does not run any model itself. It accepts a client's
26// InferRequest (prompt + max_tokens), selects a backend replica via the
27// LoadBalancer, opens a server-streaming Generate RPC to that replica,
28// and forwards each generated token back to the client as an
29// InferResponse. From the client's perspective, the gateway is a single
30// stable endpoint; the actual replica serving the request is transparent.
31//
32// Mid-stream failover (DR4):
33// If the backend replica fails or disconnects during streaming, the
34// gateway does NOT return an error immediately. Instead, it selects a
35// new replica via the load balancer and continues the stream from where
36// it left off (using tokens_already_generated to tell the new replica
37// how many tokens were already produced). The client sees a seamless
38// token stream, possibly from multiple replicas. Up to max_retries
39// failover attempts are made before giving up.
40//
41// Integration with other components:
42// - ReplicaRegistry: provides the set of known replicas and their state.
43// - LoadBalancer: selects which replica to route each request to.
44// - CircuitBreakerManager: gates requests to replicas that are returning
45// errors (application-level health, complementing gossip-level health).
46// ---------------------------------------------------------------------------
