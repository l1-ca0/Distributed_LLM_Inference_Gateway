Detailed Description

InferenceClient: blocking gRPC client for the LLM inference service.

Typical usage is against the gateway (which load-balances across replicas), but the client also exposes GenerateDirect() for bypass mode that talks to a replica directly — used by tests to verify per-replica behavior without the gateway in the loop.

All Infer* methods block the caller's thread until the stream completes, fails, or is cancelled. Each InferenceClient owns a gRPC channel to its target_address; channels are thread-safe and may be reused across sequential calls, but concurrent calls on the same InferenceClient are

not supported (use InferConcurrent for parallelism instead).
