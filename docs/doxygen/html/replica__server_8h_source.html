28// ---------------------------------------------------------------------------
29// ReplicaServer: simulated LLM inference replica backing the gateway.
30//
31// Implements the LLMReplica gRPC service, whose main RPC is a
32// server-streaming Generate: given a prompt and max_tokens, produce one
33// GenerateResponse every token_delay_ms milliseconds until max_tokens
34// tokens have been streamed. No real model is run; the purpose of this
35// class is to exercise the distributed-systems machinery around it
36// (routing, failover, backpressure, gossip), not ML.
37//
38// Threading model:
39// - A gRPC server thread pool handles each Generate call on a separate
40// thread; active_requests_ is incremented on entry and decremented on
41// exit. max_capacity_ bounds the number of concurrent streams the
42// replica advertises to the gateway; requests beyond that cap are
43// rejected with RESOURCE_EXHAUSTED so the gateway can reroute.
44// - Fault-injection state (error_rate_, reject_all_) is stored in
45// atomics so tests can flip failure modes at runtime without
46// restarting the replica.
47//
48// Shutdown:
49// Stop() is a graceful drain — it signals the server and waits for
50// in-flight streams to finish. Kill() is a hard shutdown that
51// force-cancels in-flight streams, modeling a process crash. These
52// distinct modes are what make mid-stream failover and rolling-update
53// tests exercise realistically different paths.
54// ---------------------------------------------------------------------------
