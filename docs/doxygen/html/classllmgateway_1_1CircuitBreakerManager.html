Detailed Description

CircuitBreakerManager: facade that maps replica_id -> CircuitBreaker.

Circuit breakers are created lazily on first access (GetOrCreate). All share the same Config, but each tracks its own sliding window and state independently. The manager's mutex protects the map structure (insert/lookup), while each CircuitBreaker has its own internal mutex for state transitions. This means checking/recording for different

replicas does not contend.

llmgateway::CircuitBreakerManager::CircuitBreakerManager ( )

============================================================================ CircuitBreakerManager: per-replica breaker lookup with lazy initialization.

The manager mutex is held for the duration of each public method call. This is acceptable because the critical section is short (map lookup + delegation to the individual CircuitBreaker, which also holds its own mutex briefly). For higher concurrency, the manager mutex could be replaced with a concurrent map, but at the expected replica count

(tens, not thousands) this is not a bottleneck.
