36// Thread-safe membership list with SWIM incarnation-based conflict resolution.
37//
38// Incarnation resolution rules:
39// - Higher incarnation always wins for the same member.
40// - At equal incarnation: DEAD > SUSPECT > ALIVE (stronger state wins).
41// - A member can refute a SUSPECT about itself by incrementing its incarnation.
42class MembershipList {
43public:

47 // This is the core conflict resolution logic.
48 bool ApplyUpdate(const MembershipUpdate& update);
49
50 // Mark a member as SUSPECT (called by SWIM sender on ping timeout).
51 // Uses the member's current incarnation — does NOT increment it.
52 bool MarkSuspect(const std::string& member_id);
53
54 // Mark a member as DEAD (called when suspect timer expires).

56
57 // Add a known member (e.g., seed node during bootstrap).
