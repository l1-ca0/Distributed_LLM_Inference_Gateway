17// Result of a single inference call. The tokens and replica_ids vectors
18// are index-aligned: replica_ids[i] identifies the replica that produced
19// tokens[i]. Under mid-stream failover or hedging, different indices may
20// reference different replicas — tests rely on this to verify routing
21// behavior. When success is false, error_message explains why; the tokens
22// vector may still contain a partial prefix streamed before the failure.
23struct InferResult {
24 std::vector<std::string> tokens;
25 std::vector<std::string> replica_ids;
26 bool success = false;
27 std::string error_message;
28 int request_index = -1; // for tracking order in concurrent batches
29};

31// Invoked synchronously on the caller's thread for each token received
32// during streaming. Returning false cancels the gRPC stream (via
33// ClientContext::TryCancel), causing the current Infer call to return
34// with success=false. Primarily used by tests to inject mid-stream
35// cancellation behavior.
36using TokenCallback = std::function<bool(const std::string& token,
37 const std::string& replica_id,
38 int token_index)>;

41// InferenceClient: blocking gRPC client for the LLM inference service.
42//
43// Typical usage is against the gateway (which load-balances across
44// replicas), but the client also exposes GenerateDirect() for bypass mode
45// that talks to a replica directly — used by tests to verify per-replica
46// behavior without the gateway in the loop.

48// All Infer* methods block the caller's thread until the stream completes,
49// fails, or is cancelled. Each InferenceClient owns a gRPC channel to its
50// target_address; channels are thread-safe and may be reused across
51// sequential calls, but concurrent calls on the same InferenceClient are
52// not supported (use InferConcurrent for parallelism instead).
53// ---------------------------------------------------------------------------
