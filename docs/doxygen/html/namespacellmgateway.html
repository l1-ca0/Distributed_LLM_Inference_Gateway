Detailed Description

Blocking gRPC client for the inference service.

Simulated LLM inference replica.

Orchestrates rolling replica updates: drain, stop, restart, wait for rejoin — one replica at a time, so the cluster keeps serving traffic on the remaining N-1 replicas throughout the upgrade.

Rolling update orchestrator.

Bounded FIFO backpressure queue.

In-memory registry of known replicas inside the gateway process.

Two-tier load balancer: consistent hashing for prompt-prefix affinity, falling back to weighted least-connections when the preferred replica is at capacity.

Client-facing gRPC service: the gateway's inference entry point.

Application-level health guard for replicas.

Lightweight structured logging with levels and timestamps.

Primary entry point for tests and the standalone client binary. Supports direct-replica mode, hedging, per-token callbacks, and concurrent batches.

Usage: LOG_INFO("gateway", "started on port %d", port); LOG_DEBUG("swim", "ping %s seq=%lu", peer_id.c_str(), seq); LOG_WARN("lb", "replica %s circuit open", id.c_str()); LOG_ERROR("replica", "generate failed: %s", err.c_str());

Control: Log::SetLevel(LogLevel::WARN); // suppress DEBUG and INFO Log::SetLevel(LogLevel::DEBUG); // show everything Log::SetQuiet(true); // suppress all output (for clean test output)

Complements SWIM gossip (which catches crashes) by catching "gray" failures where a replica is alive on the network but failing inference requests.

Accepts InferRequest, routes to a replica via the LoadBalancer, and streams tokens back with mid-stream failover and optional hedging.

Reads from the ReplicaRegistry for current state.

Populated from SWIM gossip callbacks and queried by the load balancer on every routing decision.

When every replica is at capacity, incoming Infer calls block on this queue until a request finishes and capacity is notified, rather than failing immediately.

Coordinates the drain → stop → restart → rejoin sequence for each replica in a rolling update. The key invariant is that at most one replica is offline at any time, so the cluster always has N-1 replicas serving traffic.

Serves the LLMReplica gRPC service, streams fake tokens at a configurable rate, and supports runtime fault injection for failover and circuit-breaker tests.
