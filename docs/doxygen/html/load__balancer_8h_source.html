17// ---------------------------------------------------------------------------
18// LoadBalancer: two-tier routing strategy for inference requests.
19//
20// Tier 1 -- Consistent Hashing (prompt-prefix affinity):
21// Hash the first 64 characters of the prompt to a point on a ring.
22// Walk clockwise until a replica with available capacity is found.
23// If the prompt prefix matches a previous request, the same replica
24// is selected. Virtual nodes spread load evenly.
25//
26// Tier 2 -- Weighted Least-Connections (capacity-aware fallback):
27// If every replica on the hash ring is at capacity (or the ring is
28// empty during startup), fall back to choosing the replica with the
29// lowest score = active_requests / weight. This prevents overload
30// and works without any prompt-specific state.
31//
32// The consistent hash ring uses virtual_nodes_ points per replica
33// (default 150) to minimize key redistribution when replicas join/leave.
34// The ring is rebuilt on membership changes (triggered by gossip events),
35// not on every request, so SelectReplica is a fast lookup.
36// ---------------------------------------------------------------------------
37class LoadBalancer {
38public:
39 explicit LoadBalancer(ReplicaRegistry& registry, int virtual_nodes = 150);
