Detailed Description

Select a replica for the given prompt.

Returns replica_id and gRPC address. Returns nullopt if no replica has capacity (caller should queue the request).

excluded lets the caller skip specific replicas — typically used by the gateway's retry loop to avoid re-selecting a replica whose circuit just tripped or whose stream just failed. Without this, consistent hashing would keep returning the same replica for the same prompt and the retry loop would exhaust attempts on a known-bad target. One replica chosen by SelectReplica. The caller typically passes grpc_address to CreateChannel and uses replica_id for registry bookkeeping (IncrementActive/DecrementActive, circuit breaker, exclude-set in retries).
